--- layout: post title: Version 0.7.0 Released ---

After more than a year since the last release, I'm happy to be able to announce the 0.7.0 release of Fleakr. There were a number of enhancements and new features in this version:

passing options to associations

Previously, when you were accessing an association for an object (e.g. user.photos) you had no control over the options passed to the underlying API call. Now you do:

{% highlight ruby %} user = Fleakr.user('the_reagent') user.photos(:per_page => 10) {% endhighlight %}

fetching a resource by its identifier

There are a few well-known web page URLs that Flickr provides for users, photos, and sets. Fleakr can consume any of these (including the short URLs) and return the correct object:

{% highlight ruby %} Fleakr.resource_from_url( 'http://www.flickr.com/people/reagent/' ) Fleakr.resource_from_url('http://flic.kr/p/7a9yQV') {% endhighlight %}

For more information, see the documentation on Fleakr.resource_from_url.

accessing exif metadata for a photo

For a given photo, you now have access to all the Exif metadata for that image:

{% highlight ruby %} photo.metadata.keys photo.metadata['Model'] {% endhighlight %}

There's more data available than just this, so check the documentation.

goodbye, active_support

ActiveSupport provided me with a quick way to get some complex functionality, but once we started seeing the release of Rails 3 it became a liability due to the incompatibility of the 2.x and 3.x releases. This became a major pain point for people wanting to use Fleakr in their Rails apps (both 2.3 and 3.0), so I removed the dependency.

other updates

In addition to these features, I made a few small tweaks:

  • Resolved issues with path names when saving a set
  • Cleaned up output when inspecting objects
  • Introduced Bundler to handle development dependencies
  • Improved support for Ruby 1.9

You can see the full list of changes on Github.
